# 두 번째 회고

## 이번 주 강의를 통해 습득하게 된 것들

이번 주에는 HTML과 CSS를 단순히 문법으로 외우는 것이 아니라,
기본적인 단어와 개념의 의미부터 이해하려고 노력했다.

예를 들어, `cat` 명령어가 concatenate의 줄임말이라는 것을 알게 되었고,
이처럼 개발에서 사용하는 용어들이 우연히 만들어진 것이 아니라
의미와 맥락을 가지고 있다는 점을 알게 되었다.

또한 img 요소, list 구조, position과 float 같은 개념을
"왜 쓰는지"를 중심으로 이해하려고 했다.

내가 분명 익숙한 영단어를 보고 있는데 컴퓨터언어에 사용되는 뜻이
다르다보니 같은 언어를 보고 있음에도 새로운 언어를 배우는 듯 했다.

## 이해가 안되거나 헷갈렸지만 어떻게 극복했는지에 대하여

처음에는 git 저장소 개념과 과제 제출 흐름이 가장 헷갈렸다.
clone, remote, push가 각각 어떤 역할을 하는지 제대로 이해하지 못해
여러 번 시도하며 막히는 상황이 있었다.

하지만 에러 메시지를 그대로 읽고,
하나씩 원인을 확인하면서 결국 문제를 해결할 수 있었다.
이 과정을 통해 문제를 피하기보다는
직접 부딪히고 이해하려는 태도의 중요성을 느꼈다.

## 달라진 생각 / 깨달음

이번 과제를 통해 프론트엔드 개발은 단순히 화면을 그리는 작업이 아니라,
정보를 어떻게 전달할 것인지에 대한 고민이라는 생각이 들었다.

특히 접근성(WAI-ARIA)을 배우면서,
보는 사람과 듣는 사람 모두에게 같은 정보를 전달하는 것이
중요하다는 점을 처음으로 깊게 생각하게 되었다.

## 내가 실제로 해본 것

아바타 과제를 진행하면서 ul/li 구조로 목록을 만들고,
figure 태그로 이미지와 상태 정보를 하나의 묶음으로 표현했다.

또한 status 점을 position으로 배치하면서,
왜 화면에 보이지 않는지 원인을 스스로 찾아보았고,
overflow와 위치 속성의 관계를 이해하게 되었다.

이 과정에서 단순히 "되게 만드는 코드"가 아니라,
"이해한 코드"를 작성하려고 노력했다.

## 다음 주에 해보고 싶은 것

다음 주에는 CSS 레이아웃에서
float, flex, grid의 차이를 직접 비교해보고 싶다.

또한 용어의 의미를 먼저 이해하는 학습 방법을 유지하면서,
새로운 개념을 만났을 때도 겁먹지 않고 접근해보고 싶다.
