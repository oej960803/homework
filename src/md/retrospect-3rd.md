# 세 번째 회고

## 이번 주 강의를 통해 습득하게 된 것들

이번 주에는 CSS를 단순히 꾸미는 용도가 아니라
레이아웃과 화면 구조를 설계하는 도구로 사용하는 방법을 배웠다.
특히 모바일 화면을 먼저 기준으로 잡고,
media query를 이용해 768px 이상에서 데스크탑 레이아웃으로 확장하는
모바일 퍼스트 반응형 웹 흐름을 이해하게 되었다.

레이아웃 구성에서는 float와 flex를 중심으로 학습했다.
float는 정렬을 위해 오래전부터 사용되던 방식이고,
flex는 여러 요소를 한 방향으로 배치하고 정렬할 때 훨씬 직관적인 방식이라는 점을
직접 비교하면서 차이를 체감할 수 있었다.

스타일 값 관리 측면에서는 CSS 변수(var)를 적극적으로 사용하는 법을 배웠다.
색상, 폰트 크기, 여백 같은 반복되는 값을 변수로 관리하니
수정이 훨씬 쉬워지고 코드 전체 흐름도 정리되는 느낌을 받았다.
여백과 글자 크기는 rem 단위를 기준으로 통일하고,
퍼센트와 픽셀 계산이 필요한 경우에는 calc() 함수를 사용해
반응형 환경에서도 깨지지 않도록 조정하는 방법을 익혔다.

## 이해가 안되거나 헷갈렸지만 어떻게 극복했는지에 대하여

이번 과제에서 가장 헷갈렸던 부분은
여러 가지 레이아웃 기법(float, flex, 반응형)을
하나의 화면 안에서 어떻게 나눠서 사용해야 하는지였다.
처음에는 그냥 화면이 맞게 나오면 된다고 생각해
flex로 전부 해결하려고 했지만,
그렇게 하면 수업에서 왜 float를 따로 다뤘는지 설명이 되지 않는다는 부분이 마음에 걸렸다.

그래서 과제 요구사항과 미리 제공된 preview 파일을 다시 천천히 확인하면서
어떤 영역에 어떤 방식이 더 적절한지를 기준으로 정리했다.
전체적인 옵션 영역 정렬에는 flex를 사용하고,
간단한 좌우 배치가 필요한 IP 보안 영역에는 float를 적용해
아직은 많이 부족하지만 각 레이아웃 기법의 쓰임새가 드러나도록 구성해 보았다.

또한 rem, calc, CSS 변수를 함께 사용하다 보니
단위와 값 계산이 헷갈리는 순간도 많았다.
이때 무작정 외우려고 하기보다는
“이 값이 화면 크기가 바뀌어도 유지되어야 하는가”를 기준으로
px, rem, calc를 구분해 사용하려고 했다.
기준을 잡고 나니 값 선택에 대한 복잡한 정리 기준이 줄었고,
코드를 수정할 때도 훨씬 안정적으로 작업할 수 있었다.

## 얼만큼 성장했는지에 대하여

이번 과제를 통해 가장 크게 성장했다고 느낀 부분은
코드를 바로 작성하기 전에 먼저 구조와 조건을 정리하는 습관이 생겼다는 점이다.
그 결과 레이아웃이 복잡해져도
전체 구조가 크게 흔들리지 않았고,
수정이 필요할 때도 특정 영역만 집중해서 고칠 수 있었다.
특히 CSS 변수를 기준으로 값을 관리하면서
디자인 수정이나 조건 변경에 훨씬 유연하게 대응할 수 있었다는 점이 인상 깊었다.
또한 단순히 결과 화면이 맞는지 여부가 아니라,
왜 이곳에는 flex를 쓰고 왜 여기에는 float를 쓰는지,
왜 rem 단위를 선택했는지 등을
스스로 조금이나마 설명할 수 있게 된 점도(완벽하게는 못 한다...) 이전과 확실히 다른 부분이다.
이 문장의 뜻이 맞는지는 잘 모르겠지만 ‘작동하는 코드’에서 ‘의도를 가진 코드’로 한 단계 넘어갔다고 느꼈다.
